Решение задачи прогнозирования целевого действия пользователя в течение 5 дней после назначения обращения.

**Метрика:** Daily Average Precision (AP)  
**Лучший результат (тест):** Daily AP = 0.51426  
**Лучший результат (валидация):** AP = 0.54749

## Описание задачи

**Цель:** предсказать, произойдёт ли успешное целевое действие в течение 5 дней после назначения обращения.

**Данные:**
- `train.csv` — обучающая выборка (13 694 строки, 119 колонок)
- `test.csv` — тестовая выборка (4 306 строк)
- `events.csv` — дополнительные события для feature engineering (254 705 строк)

**Особенности задачи:**
- Целевое событие составляет **20.75%** (дисбаланс умеренный)
- **5 типов событий:** item_view, search, favorite, chat_open, call_click
- **97 колонок с пропусками** из 119
- Тест **позже train** по времени → используется временная валидация

## Выполненные шаги

### Шаг 0: Загрузка и запуск quickstart.ipynb
- Загружены библиотеки и данные
- Запущен baseline (Logistic Regression)
- Baseline AP = 0.48506

### Шаг 1: Изучение данных (EDA)
- Размеры: train 13 694 × 119, test 4 306 × 118, events 254 705 × 7
- Доля target = 20.75%
- 7 категориальных признаков
- 5 типов событий
- 97 колонок с пропусками
- Распределение по датам: train 07.04–22.04 (16 дней), test 23.04–27.04 (5 дней)

### Шаг 2: Флаги пропусков (is_missing)
- Добавлено 97 флагов пропусков для колонок с NaN
- Итог: train (13 694, 216), test (4 306, 215)

### Шаг 3: Построение признаков из events.csv
- Фильтрация событий **до** назначения (защита от читинга): 91.8% событий доступны
- Создано **8 новых признаков**:
  - `total_events` — общее количество событий
  - `unique_event_types` — количество уникальных типов событий
  - `event_span_days` — разброс событий во времени
  - `nw_item_view`, `nw_search`, `nw_favorite`, `nw_chat_open`, `nw_call_click`
- Итог: train (13 694, 224), test (4 306, 223)

### Шаг 4: Обучение базового LightGBM
- LightGBM с early stopping (50 раундов)
- Временная валидация (70% дат — обучение, 30% — валидация)
- Категориальные признаки: преобразованы в тип `category`
- Результат: AP = 0.53832 (+11.0% к baseline)

### Шаг 5: Ансамбль моделей
- Для повышения стабильности и качества прогнозов обучен **ансамбль из 3 моделей** LightGBM с разными seed (42, 123, 456)
- Предсказания моделей усредняются
- Результат на валидации: AP = 0.54749

### Шаг 6: Анализ важности признаков
- Топ-3 важных признака: `seller_page_views_30d`, `seller_page_views_14d`, `user_age_days`
- Признаки из events попали в топ-30 → feature engineering был полезен!
